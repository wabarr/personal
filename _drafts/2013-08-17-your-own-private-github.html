  1. Set up a git application on webfaction with their 1-click installer.  More info on that here.
  2. Once you have git web application serving up central repositories, change your working directory to the folder where your repositories will live and create an empty directory mkdir test_repo.git
  3. Then cd test_repo.git and git --bare init to initialize the folder as a "bare" repository.  Git is a distributed VCS, meaning that under many use-cases, there is no centralized repository.  If you want to use git with a centralized repo, then you create a  special "bare" repo, which has no "working tree" of files like a normal repository.  You interact with a bare repo by pulling from it or pushing from it.  Finally, do git config http.receivepack true to allow http access.
  4. Now, open the client application that you want to use.  In this case, lets do Rstudio. 
  5. In Rstudio, VCS only works when you are using Rstudio "projects", rather than simply creating text files for scripts.  So lets start a new project from the "Project > Create Project" menu.  Choose the option to "check out project from version control" and then chose "git". Enter the URL to the repository on your webfaction git web application, in my case http://git.ancienteco.com/test_repo.git.
  6. Rstudio clones the remote repository into your new project folder.  In this case, there are no files in it yet, but we can change that with our local machine
  7. Create a new R script file in Rstudio, and add some code.  Then save it.
